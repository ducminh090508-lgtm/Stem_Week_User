1. ARCHITECTURAL OVERVIEW
-------------------------
The STEM Week platform is a robust, low-latency scoring and management system designed for physical/digital hybrid escape-room style competitions.

A. Server-Side (production/server/):
   - Core Engine: C++17, custom socket event loop (`poll` based).
   - Database: SQLite 3.4x (Relational storage for ACID compliance).
   - Security Layer: OpenSSL 3.x (TLS 1.3 encryption).
   - Key Files:
     * src/server.cpp: Low-level socket management and TLS wrapping.
     * src/main.cpp: Command orchestration, API definitions, and logic.
     * include/byteBuffer.h: High-performance memory management for incoming packets.

B. Client-Side (production/user-app/):
   - TUI Engine: Textual (Async Python framework with CSS-like styling).
   - Core Modules:
     * app/interface/dashboard.py: Entry portal (Fetch-style landing).
     * app/interface/question.py: Main hub for subject selection.
     * app/functions/question.py: Network bridge (WebSockets).

3. DATABASE SCHEMA (SQLite3)
----------------------------
The system uses a relational schema to track progress across multiple rotations.

- Teams: (team_id, team_name, current_rotation_id, status)
- TeamRotations: Tracks (start_time, end_time, status) per team/rotation.
- Submissions: Immutable log of every answer attempt (is_correct, timestamp).
- Penalties: Log of manual/auto time additions (HINT: 300s, INACCURATE: 30s).
- Leaderboard (View): Aggregates (End - Start) + SUM(Penalties).

4. WEBSOCKET API REFERENCE
--------------------------
Port: 8080 (Encrypted WSS)

A. TEAM OPERATIONS:
   - SUBMIT <tid> <subId> <ans> -> Returns "CORRECT" | "INCORRECT"
   - GET_STATUS <tid>           -> Returns Progress JSON
   - GET_LEADERBOARD            -> Returns Sorted Rank Array

B. ADMIN OPERATIONS (Requires ADMIN_LOGIN):
   - START/PAUSE/FINISH <ids>   -> Controls global/team timers.
   - PENALTY <ids> <type> <sec> -> Adds time to score.
   - SET_WIN <tid> <bool>       -> Manual winner override.

5. RESTful API ENDPOINTS (HTTPS)
--------------------------------
- GET /api/leaderboard: Raw JSON array of rankings.
- GET /api/debug: Full system state dump (For admin monitoring).
- GET /api/logs: Live tail of the last 100 system events.

6. BUILD & DEPLOYMENT
---------------------
- Server: Built using CMake 3.10+.
  Steps: `mkdir build; cd build; cmake ..; cmake --build .`
- User App: Distributed as a PyInstaller binary or run via `python main.py`.
- Certificates: Self-signed RSA 2048-bit (located in /cert).
